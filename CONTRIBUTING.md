# Contributing to Kiro-Go Plus

Everyone may download, fork, inspect, and propose improvements to this public
repository.

## Change Process

1. Fork the repository and create a feature branch.
2. Keep account credentials, tokens, local runtime data, and personal
   information out of commits.
3. Run `go test ./...` before opening a pull request.
4. Open a pull request targeting `main`.
5. Wait for approval from the repository code owner before merging.

The protected `main` branch requires a pull request and code-owner approval for
non-admin changes. The maintainer may reject changes that are unsafe,
out-of-scope, insufficiently tested, or incompatible with the project's goals.

## 提交修改

所有人都可以下载、Fork、查看并为本公开仓库提出改进。

1. Fork 仓库并创建功能分支。
2. 不得在提交中包含账号凭证、Token、本地运行数据或个人信息。
3. 提交 Pull Request 前运行 `go test ./...`。
4. Pull Request 必须以 `main` 为目标分支。
5. 修改获得仓库代码所有者批准后才能合并。

受保护的 `main` 分支要求非管理员修改必须通过 Pull Request 和代码所有者
审批。维护者可拒绝存在安全风险、超出项目范围、测试不足或与项目目标不兼容
的修改。
